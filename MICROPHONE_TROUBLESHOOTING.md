# 麦克风权限和连接故障排除指南

## 🎤 常见问题和解决方案

### 问题 1: "麦克风权限被拒绝"

**症状**:
```
语音连接失败
麦克风权限被拒绝。请在浏览器设置中允许访问麦克风。
```

**原因**:
- 用户在浏览器权限提示中点击了"拒绝"
- 浏览器设置中禁用了麦克风访问权限

**解决方案**:

#### Chrome/Edge:
1. 点击地址栏左侧的锁形图标
2. 找到"麦克风"或"Microphone"选项
3. 将其设置为"允许"
4. 刷新页面并重试

#### Firefox:
1. 点击地址栏左侧的信息图标
2. 找到"麦克风"权限
3. 将其设置为"允许"
4. 刷新页面并重试

#### Safari:
1. 打开"系统偏好设置" → "安全与隐私" → "隐私"
2. 选择"麦克风"
3. 确保 Safari 在列表中并被勾选
4. 刷新页面并重试

---

### 问题 2: "未找到麦克风设备"

**症状**:
```
语音连接失败
未找到麦克风设备。请检查硬件连接。
```

**原因**:
- 电脑没有连接麦克风
- 麦克风驱动程序未安装或损坏
- 麦克风被禁用

**解决方案**:

#### Windows:
1. 右键点击任务栏的音量图标
2. 选择"打开声音设置"
3. 向下滚动到"输入"部分
4. 确保麦克风已连接并被识别
5. 如果显示"未连接"，检查硬件连接

#### Mac:
1. 点击苹果菜单 → "系统偏好设置"
2. 选择"声音" → "输入"
3. 确保麦克风在列表中并被选中
4. 检查输入音量不是静音

#### Linux:
```bash
# 检查麦克风是否被识别
arecord -l

# 检查 PulseAudio 状态
pactl list sources
```

### 问题 3: "麦克风被其他应用占用"

**症状**:
```
语音连接失败
麦克风被其他应用占用。请关闭其他使用麦克风的应用。
```

**原因**:
- 其他应用正在使用麦克风（如 Zoom、Skype、Discord 等）
- 浏览器的另一个标签页正在使用麦克风

**解决方案**:

1. **关闭其他应用**:
   - 关闭 Zoom、Skype、Discord 等通话应用
   - 关闭其他使用麦克风的应用

2. **检查浏览器标签页**:
   - 关闭其他可能使用麦克风的标签页
   - 刷新当前页面

3. **重启麦克风**:
   - 拔出麦克风再插回
   - 重启电脑

---

### 问题 4: "WebSocket 连接失败"

**症状**:
```
语音连接失败
WebSocket 连接失败
```

**原因**:
- 后端服务未运行
- 网络连接问题
- 防火墙阻止了连接
- WebSocket URL 配置错误

**解决方案**:

1. **检查后端服务**:
   ```bash
   # 检查后端是否运行
   curl http://localhost:7080/api/health
   ```

2. **检查网络连接**:
   - 确保网络连接正常
   - 尝试访问其他网站确认网络可用

3. **检查防火墙**:
   - 允许浏览器访问网络
   - 检查防火墙是否阻止了 WebSocket 连接

4. **检查 WebSocket URL**:
   - 打开浏览器开发者工具 (F12)
   - 查看 Console 标签
   - 确认 WebSocket URL 是否正确

---

### 问题 5: "连接中..." 一直不连接

**症状**:
```
对话中 — 连接中...
```
（状态一直停留在"连接中"）

**原因**:
- 后端服务响应缓慢
- 网络延迟高
- 后端服务配置不完整

**解决方案**:

1. **检查后端日志**:
   ```bash
   # 查看后端是否有错误
   # 检查 REALTIME_API_KEY 等环境变量是否设置
   ```

2. **增加超时时间**:
   - 等待更长时间（通常 5-10 秒）
   - 如果仍然失败，检查后端日志

3. **重启后端**:
   ```bash
   # 停止后端
   # 重新启动后端
   go run cmd/server/main.go
   ```

## 📋 诊断步骤

### 1. 检查浏览器控制台

打开浏览器开发者工具 (F12)，查看 Console 标签：

```javascript
// 检查麦克风权限状态
navigator.permissions.query({ name: 'microphone' })
  .then(result => console.log(result.state))

// 检查 getUserMedia 支持
console.log(!!navigator.mediaDevices?.getUserMedia)
```

### 2. 测试麦克风

在浏览器控制台运行：

```javascript
// 测试麦克风访问
navigator.mediaDevices.getUserMedia({ audio: true })
  .then(stream => {
    console.log('✅ 麦克风可用')
    stream.getTracks().forEach(track => track.stop())
  })
  .catch(err => console.error('❌ 麦克风错误:', err.name, err.message))
```

### 3. 检查网络连接

```javascript
// 检查 WebSocket 连接
const ws = new WebSocket('ws://localhost:7080/api/ws/realtime/ai-interview')
ws.onopen = () => console.log('✅ WebSocket 连接成功')
ws.onerror = (err) => console.error('❌ WebSocket 错误:', err)
```

## 🛠️ 高级故障排除

### 启用详细日志

在浏览器控制台中：

```javascript
// 启用详细日志
localStorage.setItem('DEBUG', '*')
// 刷新页面
location.reload()
```

### 检查麦克风权限历史

#### Chrome:
1. 打开 `chrome://settings/content/microphone`
2. 查看允许和阻止的网站列表
3. 如果网站在阻止列表中，点击删除

#### Firefox:
1. 打开 `about:preferences#privacy`
2. 向下滚动到"权限"部分
3. 点击"麦克风"旁的"设置"
4. 查看并删除阻止的网站

## 🎯 总结

| 错误信息 | 主要原因 | 快速解决 |
|---------|--------|--------|
| 权限被拒绝 | 用户拒绝权限 | 在浏览器设置中允许麦克风 |
| 未找到设备 | 没有麦克风 | 连接麦克风或检查驱动 |
| 被占用 | 其他应用使用 | 关闭其他应用 |
| WebSocket 失败 | 后端问题 | 检查后端服务 |
| 一直连接中 | 后端响应慢 | 等待或重启后端 |

**大多数问题可以通过允许麦克风权限和检查网络连接来解决！** 🎉
